Trust Wallet parses the EIP-681 /transfer function parameters for display but executes the raw data parameter as transaction calldata. This lets you show 1 USDT (via uint256=1000000) while executing 4+ USDT (via data containing ABI-encoded calldata).

Format:
ethereum:0xTOKEN@1/transfer?address=RECIPIENT&uint256={displayAmount}&data=0xa9059cbb...

How it works: The data field contains the full ABI-encoded calldata (function selector + padded address + padded amount). Trust Wallet shows the declared uint256 amount in the confirmation screen, but sends the data as raw calldata — which encodes a different amount. Tested on iOS & Android — confirmed working.
