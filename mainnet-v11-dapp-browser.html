🔬 v11 — Dapp Browser Attack Surface

New angle: trust://browser opens a malicious dapp that requests a different TX than it displays.

Dead end on deep links: value=0/1/any doesn't help — wallet pre-checks USDT contract and disables confirm. erc20_transfer (only confirmable path) ignores data. Moving to dapp browser approach.

How the dapp browser attack works

  1. QR code opens trust://browser?url=DATA_URL which launches Trust Wallet's built-in browser
  2. The page is a malicious dapp that loads and connects to wallet via ethereum.request()
  3. Dapp shows harmless UI text but requests a transaction with our drain calldata
  4. Wallet shows transaction request — user confirms → 4 USDT transfers
