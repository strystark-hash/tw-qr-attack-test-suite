🔬 v14 — Dapp Browser Attack

Fallback if EIP-681 data override is blocked — uses trust://browser + Web3 eth_sendTransaction

⚡ How This Works

EIP-681 deep links with data param appear to be stripped by modern Trust Wallet (erc20_transfer has no data field). New angle: trust://browser?url=... opens Trust Wallet's built-in dapp browser. The loaded dapp page uses window.ethereum.request({method: 'eth_sendTransaction', ...}) to send raw transactions with full control over data.

Trick: The dapp shows fake UI ("Claim 0.1 ETH Airdrop") but requests a USDT transfer transaction. The wallet confirmation screen shows the REAL transaction (to: USDT, data: transfer 4 USDT), bypassing the deep link parser entirely.
