📌 Key Findings

❌ ERC20Transfer has NO data field

Wallet-core proto for token transfers has no bytes data field. Any data in EIP-681 URI is stripped during transaction building. This is the root cause of all failures.

✅ Transfer/ContractGeneric HAVE data fields

Native ETH transfer and generic contract interaction protos both carry bytes data. But when these routes are triggered, the wallet shows ETH amounts (not USDT), defeating the display mismatch.

⚠️ v5/C — Closest to Working

ethereum:0xUSDT@1/drain?address=RCP&uint256=1M&data=... — transaction EXECUTED ($0.04 gas paid) but data was stripped → no USDT moved. Proves confirm works for generic paths.
