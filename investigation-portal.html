🔍 Key Findings

❌ ERC20Transfer has NO data field

Wallet-core proto for token transfers has no bytes data. Any data in EIP-681 URI is stripped during tx building. Root cause of all failures.

✅ Transfer/ContractGeneric HAVE data fields

Native ETH and generic interaction protos both carry bytes data. But when these routes trigger, wallet shows ETH amounts — not USDT.

⚠️ Closest: v5/C

/drain?addr=RCP&uint256=1M&data=... — transaction executed ($0.04 gas paid) but data stripped → no USDT moved. Proves confirm works for generic paths.

💡 Dapp Browser = Best Remaining

trust://browser?url=... opens dapp browser with Web3 provider. Dapp calls eth_sendTransaction directly — no EIP-681 parser, no data loss. UNTESTED.

Architecture: Wallet-Core Routing

URI → EIP681Parser
├─ Has `amount`? → ERC20Transfer (data ignored)
├─ Has `/transfer`+addr+uint256? → ERC20Transfer (data ignored)
├─ Has `/approve`+addr+uint256? → ContractGeneric (data MAY pass)
├─ Has `value` (no amount)? → Transfer (data passes, shows ETH)
└─ Has `data` only? → ContractGeneric (data passes)
